# Prompt: Build Hero + Bento Grid Sections

Create a new React application with a **fullscreen scroll-triggered video Hero section** and a **Bento Grid "About" section**.

---

## **1. Pre-requisite: Frame Generation**
**IMPORTANT:** The `public` folder currently contains a video file `hero-video.mp4`. You must first convert this video into a sequence of optimized WebP frames to ensure smooth playback and fast loading.

**Run this FFmpeg command in your terminal before building:**
```bash
mkdir -p public/frames
ffmpeg -i public/hero-video.mp4 -vf "scale=1920:1080" -c:v libwebp -quality 85 -compression_level 6 -start_number 1 public/frames/frame_%04d.webp
```
*Note: This generates 1080p frames with quality 85. This visual standard balances high quality (~70KB/frame) with performance, preventing the "heavy load" stuttering issues.*

## **4. SECTION 1: Hero (Fullscreen Scroll-Triggered Video)**

### **Behavior:**
- Container height: `800vh` (creates scroll space)
- Sticky viewport: `100vh` fixed canvas inside
- On scroll: Animate through the **generated sequence of WebP frames** drawn on a `<canvas>` element using GSAP ScrollTrigger.
- **Progressive Batch Loading:**
  - **CRITICAL:** Do NOT wait for all frames to load.
  - Load the first **30 frames** immediately.
  - Once 30 are loaded,fade out the poster and unlock the scroll animation.
  - Load the remaining frames in background batches (e.g., chunks of 30) to prevent UI freezing.
- On mobile: Ignore height-only resize events (address bar toggle) to prevent jitter.

- Black background
- Poster image visible until first batch loads.
- Overlay layers:
  - Global darken: `bg-black/10`
  - Vignette: `radial-gradient(circle, transparent 50%, rgba(0,0,0,0.4) 100%)`
  - Bottom gradient: `from-black/40`
- CSS filter on canvas container: `saturate(1.25)`

### **Content:**
- Top-right: LinkedIn and Email links (vertical stack)
- Bottom-left: Large title "AI/ML Engineer" (responsive text size)
- Bottom-right: Short description + animated scroll indicator (mouse icon on mobile, arrow on desktop)
- All text white, Syne/Satoshi fonts

### **Responsive:**
- Mobile: Single column, centered text, smaller fonts
- Desktop: Flex layout with `justify-between`

## **8. Key Implementation Notes**
1. **Frame Generation**: Ensure the `ffmpeg` command (Step 1) is run first. The app depends on `public/frames/frame_XXXX.webp`.
2. **Performance**: Use the optimized 1080p frames. Do NOT use the raw video file directly in the canvas.
3. **Smooth Scroll**: Implement Lenis at the App level.
4. **GSAP**: Use `useLayoutEffect` or `useEffect` with cleaning up ScrollTriggers.
5. **Caching**: Append `?v=1` to frame URLs to prevent caching of old assets if you regenerate them.
6. **Bento Grid Layout**: Use standard CSS Grid properties (`col-span`, `row-span`) for all scaling and positioning. **Do NOT use negative margins** (e.g., `-mt-`) to force overlaps or position cards, as this breaks responsiveness.
7. **Animation Robustness**:
   - Use `ScrollTrigger.batch()` for grid items to ensure they animate even if rapid scrolling occurs or layout shifts.
   - Sync **Lenis** with **ScrollTrigger** by adding `lenis.on('scroll', ScrollTrigger.update)` and hooking `lenis.raf` into `gsap.ticker`.
   - Always ensure initial state (`opacity: 0`) is set via `gsap.set()` before animation to prevent FOUC or "black screen" issues.
